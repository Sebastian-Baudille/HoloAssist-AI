# Isaac Sim RL Migration Plan

Plan for porting the UR3e + RG2 reach/pick PPO pipeline from Gazebo
([ur3e_rl_ws/](ur3e_rl_ws/)) to **NVIDIA Isaac Sim + Isaac Lab** on native Windows 11.

---

## Why migrate

| | Current (Gazebo) | Target (Isaac Lab) |
|---|---|---|
| Parallel envs | 4 (CPU-bound ceiling per [TRAINING_NOTES.md](ur3e_rl_ws/TRAINING_NOTES.md)) | 1024–4096 (GPU) |
| Throughput | ~hundreds steps/s | 50k–100k+ steps/s |
| 200 k-step run | ~30 min | < 1 min |
| ROS in training loop | yes (rclpy + bridges) | no — pure GPU tensor ops |
| Sim-to-real tools | manual | built-in domain randomisation |
| Setup cost | done | ~1 day install + ~1 week port |

The bottleneck today is throughput. Isaac Lab removes it.

---

## Locked-in decisions

| Decision | Choice | Why |
|---|---|---|
| GPU | RTX 30/40/50-series (Windows host) | Required for PhysX GPU + RTX sensors |
| Platform | Native Windows 11 | Isaac Sim runs natively; avoids WSL display/GPU quirks |
| RL framework | **RSL-RL** (Isaac Lab default) | Fully GPU-vectorised PPO; fastest with thousands of envs |
| Observation | Ground-truth low-dim | 6 joint pos + 6 joint vel + 7 TCP pose + 3 cube pos = 22-D. Matches existing `OBSERVATION_SIZE` layout from [ros_interface.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/ros_interface.py) |
| Env API | `DirectRLEnv` | 1:1 port of existing Gym env [ur3e_pick_place_env.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/envs/ur3e_pick_place_env.py) |
| Algorithm | PPO | Keep current algorithm; reuse hyperparameter intuition |

Not picked yet: behaviour-cloning warm-start ([pretrain_from_demos.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/pretrain_from_demos.py)) port — RSL-RL doesn't support BC out of the box; revisit after a baseline policy works.

## Phased plan

### Phase 0 — Host prerequisites
- NVIDIA driver ≥ 537.58 (`nvidia-smi`)
- Visual C++ Redistributable 2015–2022
- Git for Windows
- ≥ 100 GB free SSD space (Isaac shader cache ≈ 20 GB)

### Phase 1 — Miniforge
Install Miniforge3 for Windows, then in PowerShell:
```powershell
conda init powershell
```

### Phase 2 — Isaac Sim
```powershell
conda create -n holo_rl python=3.10 -y
conda activate holo_rl
pip install --upgrade torch==2.5.1 torchvision==0.20.1 --index-url https://download.pytorch.org/whl/cu121
pip install isaacsim[all,extscache]==4.5.0 --extra-index-url https://pypi.nvidia.com
python -c "from isaacsim import SimulationApp; app = SimulationApp({'headless': True}); app.close()"
```
First import triggers a one-time shader cache build (~10 min). Do not interrupt.

### Phase 3 — Isaac Lab
```powershell
cd C:\Users\sebas\Github
git clone https://github.com/isaac-sim/IsaacLab.git
cd IsaacLab
.\isaaclab.bat --install rsl_rl
.\isaaclab.bat -p scripts\reinforcement_learning\rsl_rl\train.py `
    --task Isaac-Cartpole-v0 --num_envs 64 --headless
```
Cartpole smoke test must produce scrolling PPO logs and an `outputs/` folder.

### Phase 4 — Project layout
Create `isaaclab_ws/` per the tree above. Make it editable-installable:
```powershell
cd isaaclab_ws
pip install -e .
```

### Phase 5 — URDF → USD
1. In WSL Ubuntu (xacro needs ROS):
   ```bash
   source /opt/ros/humble/setup.bash
   xacro ur3e_rl_ws/src/ur3e_gazebo_sim/urdf/ur3e_rg2_benchtop.urdf.xacro \
         > /tmp/ur3e_rg2.urdf
   ```
2. Copy URDF (and referenced meshes — keep paths absolute or co-located) into
   `isaaclab_ws/holoassist_tasks/assets/`.
3. Convert in Windows PowerShell:
   ```powershell
   .\IsaacLab\isaaclab.bat -p scripts\tools\convert_urdf.py `
       <abs path>\ur3e_rg2.urdf `
       <abs path>\ur3e_rg2.usd `
       --merge-joints --fix-base
   ```
4. Verify articulation: 6 UR3e revolute joints + 2 RG2 finger joints, base fixed,
   meshes visible in the GUI viewer.

### Phase 6 — Env config (`ur3e_reach_env_cfg.py`)
- `ArticulationCfg` → `ur3e_rg2.usd`; actuator stiffness 400, damping 40
- `RigidObjectCfg` cube: 4 cm, 0.05 kg, μ=0.8 (matches [CLAUDE.md](CLAUDE.md) sim facts)
- Table: static rigid body or prim
- `sim.dt = 1/120`, `decimation = 4` → 30 Hz policy
- `episode_length_s ≈ 6.7` (= 200 × dt × decimation)
- `action_space = 6`, `observation_space = 22`

### Phase 7 — Env class (`ur3e_reach_env.py`)
Subclass `omni.isaac.lab.envs.DirectRLEnv`. Port mapping:

| `DirectRLEnv` method | Source (existing repo) |
|---|---|
| `_setup_scene` | scene from [ur3e_pick_place_world.launch.py](ur3e_rl_ws/src/ur3e_gazebo_sim/launch/ur3e_pick_place_world.launch.py) |
| `_pre_physics_step` / `_apply_action` | action scaling in [ur3e_pick_place_env.py:68-73](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/envs/ur3e_pick_place_env.py#L68-L73) |
| `_get_observations` | `build_observation` in [ros_interface.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/ros_interface.py) |
| `_get_rewards` | `compute_reward` in [reward.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/reward.py) |
| `_get_dones` | `check_failure` / `check_success` in [reward.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/reward.py) |
| `_reset_idx` | randomisation ranges in [ur3e_pick_place_env.py:33-36](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/envs/ur3e_pick_place_env.py#L33-L36) |

Rules during the port:
- All ops are `torch` on `self.device`, batched over `num_envs`.
- Drop `rclpy`, `RosInterfaceNode`, `wait_until_ready`, any `time.sleep`.
- Drop `SafetyChecker` from the inner loop. Express joint limits as `clip` on
  the target; TCP-height violation becomes a reward penalty + termination.

### Phase 8 — Task registration
- `ur3e_reach/__init__.py`: `gym.register("HoloAssist-UR3e-Reach-v0", ...)`
- `holoassist_tasks/__init__.py`: import `ur3e_reach` to trigger registration
- `agents/rsl_rl_ppo_cfg.py`: PPO config based on Isaac Lab's
  `Isaac-Reach-Franka-v0` (lr 3e-4, γ 0.99, clip 0.2, entropy 0.01)

### Phase 9 — Train
```powershell
conda activate holo_rl
cd isaaclab_ws
python ..\IsaacLab\scripts\reinforcement_learning\rsl_rl\train.py `
    --task HoloAssist-UR3e-Reach-v0 --num_envs 4096 --headless
```
- 4090: ~50–100 k steps/s
- 5–30 M steps to convergence (10–60 min wall time)
- Drop `--num_envs` to 1024 on 8 GB cards or on OOM
- Omit `--headless` only for live debugging (slow)

### Phase 10 — Monitor
```powershell
tensorboard --logdir logs/rsl_rl/holoassist_ur3e_reach
```
Watch `Train/mean_reward`, `Episode/success_rate`. Flat reward > 2 M steps →
same triage as [TRAINING_NOTES.md](ur3e_rl_ws/TRAINING_NOTES.md) section
"Robot speed" (action scale, reachability, reward shaping).

### Phase 11 — Eval + export
```powershell
python ..\IsaacLab\scripts\reinforcement_learning\rsl_rl\play.py `
    --task HoloAssist-UR3e-Reach-v0 --num_envs 16 `
    --checkpoint logs\rsl_rl\holoassist_ur3e_reach\<run>\model_<N>.pt
```
Produces `exported/policy.pt` (TorchScript) and `exported/policy.onnx` —
the deployment artifacts.

### Phase 12 — Deploy to ROS
Adapt [rl_policy_node.py](ur3e_rl_ws/src/ur3e_policy_controller/ur3e_policy_controller/rl_policy_node.py):
```python
import onnxruntime as ort
session = ort.InferenceSession("policy.onnx", providers=["CPUExecutionProvider"])
action = session.run(None, {"obs": obs.astype(np.float32)})[0]
```
**Hard requirement**: the observation vector built by [ros_interface.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/ros_interface.py) must match
`_get_observations` byte-for-byte (order, units, frames). Assert
`OBSERVATION_SIZE` at both ends. This is the #1 silent failure mode.

Test order:
1. Replay Isaac-Lab observations through ROS node → action sanity check
2. Run policy in existing Gazebo sim ([ur3e_pick_place_world.launch.py](ur3e_rl_ws/src/ur3e_gazebo_sim/launch/ur3e_pick_place_world.launch.py))
3. Run on real UR3e with `SafetyChecker` enabled, `max_delta_rad=0.05`

### Phase 13 — Sim-to-real (later)
Domain randomisation in `_reset_idx`:
- Joint friction / damping ±20 %
- Cube mass ±30 %, μ ∈ [0.5, 1.0]
- Action noise σ = 0.01 rad
- Joint pos noise σ = 0.001 rad; cube pos noise σ = 0.005 m (matches
  D435i depth noise from [CLAUDE.md](CLAUDE.md))

## What stays, what's replaced

**Stays:**
- [reward.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/reward.py) logic (tensorised)
- [ros_interface.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/ros_interface.py) observation layout (mirrored in `_get_observations`)
- [SafetyChecker](ur3e_rl_ws/src/ur3e_safety_layer/ur3e_safety_layer/safety_checker.py) for deployment only
- Gazebo sim ([ur3e_gazebo_sim/](ur3e_rl_ws/src/ur3e_gazebo_sim/)) as a sim-to-real intermediate test bench
- [rl_policy_node.py](ur3e_rl_ws/src/ur3e_policy_controller/ur3e_policy_controller/rl_policy_node.py) (model loader swapped to ONNX Runtime)

**Replaced:**
- [ur3e_pick_place_env.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/envs/ur3e_pick_place_env.py) → `ur3e_reach_env.py` (DirectRLEnv)
- [train_ppo.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/train_ppo.py) / [train_ppo_parallel.py](ur3e_rl_ws/src/ur3e_rl_env/ur3e_rl_env/train_ppo_parallel.py) → `IsaacLab/scripts/reinforcement_learning/rsl_rl/train.py`
- SB3 PPO → RSL-RL PPO
- 4-Gazebo SubprocVecEnv → 4096-env GPU vec
- MoveIt collision checking during training → reward penalty + termination

**Open questions** (revisit after baseline works):
- Behaviour-cloning warm-start: needs an RSL-RL fork or switch to SKRL/SB3 adapter
- Camera-based observation: defer until ground-truth policy is solid
- Pick-and-place vs reach-only: current Gazebo env is reach-only; extending to
  full pick-and-place needs RG2 gripper actuation in the env + reward redesign
