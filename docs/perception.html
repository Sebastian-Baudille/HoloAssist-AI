Overview

Perception Pipeline

The perception stage captures RGB-D point clouds from a simulated Intel RealSense D435i camera running in Gazebo Fortress. Point clouds are published on /camera/points and saved to disk as binary PLY files by save_pointcloud.py.

Camera Configuration

All camera parameters are defined in sim_params.yaml:

  • Resolution: 640 × 480
  • FOV: configurable via horizontal_fov_deg
  • Depth noise: Gaussian σ = 0.005 m
  • Update rate: configurable, default matches Gazebo sim rate
  • Pose: 0.528 rad (~30°) downward tilt

Camera-to-World Transform

Point clouds are published in the camera body frame (+X = look direction). Before any height-based operations, they must be transformed to the world frame using the static TF chain:

map → d435i_link → d435i/d435i_link/rgbd

PLY Output

Files are saved to ~/holoassist_pointclouds/ with automatic versioning:

<description>_<N>cubes_<size>mm_v<NNN>.ply
