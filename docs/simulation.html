Gazebo — Ollie Lau & John Chen

Gazebo Ignition Fortress

The Gazebo simulation was bootstrapped by Ollie Lau (Gazebo Classic, UR3e + table, initial PPO scripts, safety layer) and extended by John Chen (ported to Ignition Fortress, Moveit2 collision avoidance, cube randomization, parallel IGN_PARTITION isolation, OnRobot RG2 integration). After handoff, Ollie owns the Gazebo sim and MoveIt2 subsystem.

What Gazebo provides

  • Simulated Intel RealSense D435i camera → /camera/points for the perception pipeline
  • UR3e + OnRobot RG2 gripper in physics simulation
  • Per-episode cube randomization (continuous uniform X/Y range)
  • Safety layer: joint limit checking, MoveIt2 collision detection
  • Parallel worker support via IGN_PARTITION environment variable

Safety layer (Ollie)

The ur3e_safety_layer package enforces max_delta_rad = 0.24 rad/step (75% of UR3e max speed at control_dt=0.1s) and publishes a collision flag from MoveIt2. The safety node monitors TCP pose and joint states, stopping the arm if a violation is detected.

MuJoCo — John Chen

MuJoCo fast training environment

John built a MuJoCo MJCF environment that exactly mirrors the Gazebo scene but runs on CPU at ~1500 steps/s with no ROS dependency. This is used for fast reward shaping and staged policy design before committing expensive GPU training runs in Isaac Sim.

Scene design

  • Floor, table at (-0.031, 0, 0.29) — matches Gazebo launch parameters
  • 4 free-body cubes with realistic physics
  • 2 bin sites for pick-and-place target
  • 6 position actuators (shoulder_pan, shoulder_lift, elbow, wrist_1, wrist_2, wrist_3)
  • RG2 gripper: all fixed joints, virtual gripper_closed boolean — no physics actuation needed for RL
  • Table top at z=1.07 m = TABLE_TOP_Z constant shared with Gazebo env
Weight transfer guarantee: The MuJoCo env observation and action spaces are defined identically to the Gazebo env. A policy trained in MuJoCo can be loaded directly into Gazebo or exported via ONNX to Isaac Sim with no architecture changes.

Assets

Robot Asset Pipeline

The UR3e + RG2 robot is modelled as a layered USD asset:

isaac_rl/assets/ ├── meshes/ 22 mesh files (14 UR3e .dae/.stl + 8 RG2 .stl) ├── urdf/ │ ├── ur_onrobot.urdf raw xacro flatten │ └── ur_onrobot_prepared.urdf mimic-stripped, URIs rewritten └── usd/ └── ur_onrobot_prepared/ layered USD ├── ur_onrobot_prepared_base.usd ├── ur_onrobot_prepared_physics.usd ├── ur_onrobot_prepared_robot.usd └── ur_onrobot_prepared_sensor.usd

URDF Preparation

prepare_urdf.py strips mimic joints and rewrites mesh URIs for Isaac Sim import. The resulting URDF is imported into USD via the Isaac Sim asset converter.

Environment

Isaac Lab Environment

The reach task uses Isaac Lab's DirectRLEnv API with modular strategy pattern:

holoassist_tasks/tasks/direct/reach/ ├── __init__.py task registration + env config ├── reach_env.py DirectRLEnv subclass ├── reach_env_cfg.py scene, robot, action, obs configs ├── actions/ │ └── joint_delta.py per-joint scaled delta actions ├── observations/ │ └── ground_truth_12d.py joint pos + EE pos + target pos └── rewards/ ├── dense_reach.py distance-based dense reward ├── dense_reach_v1.py + convergence bonus └── dense_reach_v2.py + action penalty
