Overview

System Overview

HoloAssist-AI is the AI extension to the RS2 HoloAssist project, built for the 41118 Artificial Intelligence in Robotics course. It adds an end-to-end perception and control pipeline: an Intel RealSense D435i captures RGB-D point clouds, DBSCAN clustering extracts 3D cube centroids with ~1.6 cm accuracy, and a PPO policy trained in NVIDIA Isaac Sim 5.1 + Isaac Lab (4096 GPU-parallel envs) controls a UR3e arm to pick and place objects.

Quick Start

Quick Start

Clone, run setup, and launch the simulation:

# One-command setup git clone https://github.com/Sebastian-Baudille/HoloAssist-AI.git cd HoloAssist-AI chmod +x setup.sh && ./setup.sh # Terminal 1 — Gazebo + RViz source ros2_ws/install/setup.bash ros2 launch holoassist_sim sim.launch.py # Terminal 2 — Capture a point cloud source ros2_ws/install/setup.bash ros2 run holoassist_sim save_pointcloud.py \ --params ros2_ws/src/holoassist_sim/config/sim_params.yaml # Terminal 3 — Run DBSCAN clustering source clustering/.venv/bin/activate python clustering/detect_cubes.py

See LAUNCH.md for the full command reference including RL training and parallel workers.

Architecture

Architecture

D435i Camera (sim or real) ↓ ROS 2: /camera/points Point Cloud Capture ──→ binary PLY on disk ↓ DBSCAN Clustering ──→ 3D centroids (k, 3) ↓ Isaac Sim 5.1 + Isaac Lab (4096 GPU-parallel envs) ↓ RSL-RL PPO Policy (DirectRLEnv → joint-delta actions) ↓ UR3e + RG2 (USD asset, reach → grasp → place)
